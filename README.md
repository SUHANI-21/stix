# Cyber Threat Intelligence Dataset using STIX 2.1

## Overview

This repository contains Cyber Threat Intelligence (CTI) data exported in **STIX 2.1 JSON format** using OpenCTI integrated with MITRE ATT&CK threat intelligence feeds.

The dataset consists of multiple CTI entities such as attack patterns, intrusion sets, malware, tools, indicators, vulnerabilities, and the relationships connecting them.

The primary objective of this repository is to provide structured threat intelligence data that can be used for:

- Threat intelligence analysis
- Knowledge graph construction
- Graph database research
- Machine learning applications
- Threat hunting
- ATT&CK mapping
- Security analytics
- Cybersecurity research projects

---

# Dataset Contents

The repository contains structured STIX 2.1 objects including:

| STIX Object Type | Description |
|---|---|
| `attack-pattern` | Adversary techniques and tactics |
| `intrusion-set` | Threat groups / APT groups |
| `malware` | Malicious software |
| `tool` | Tools used during attacks |
| `campaign` | Threat campaigns |
| `indicator` | Indicators of compromise (IOCs) |
| `vulnerability` | Security vulnerabilities / CVEs |
| `relationship` | Links between entities |

---

# Data Source

The dataset was generated by:

1. Running OpenCTI using Docker
2. Connecting OpenCTI with MITRE ATT&CK feeds
3. Exporting threat intelligence objects in STIX 2.1 JSON format

---

# STIX 2.1 Format

The dataset follows the STIX 2.1 specification for representing cyber threat intelligence.

Example STIX object:

```json
{
  "type": "attack-pattern",
  "id": "attack-pattern--123456",
  "name": "Phishing",
  "description": "Adversaries send fraudulent messages..."
}
```

Example relationship object:

```json
{
  "type": "relationship",
  "relationship_type": "uses",
  "source_ref": "intrusion-set--111",
  "target_ref": "malware--222"
}
```

Relationship meaning:

```text
Intrusion Set → uses → Malware
```
